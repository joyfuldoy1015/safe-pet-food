# 집사력 테스트 UX/UI 개선 제안

분석일: 2026-04-06

---

## 1. 랜딩 (Landing)

**문제점:**
- 배경 플로팅 이모지 8개가 랜덤하게 흩어져 있는 방식이 전형적인 "바이브코딩 패턴" — 시각적 목적 없이 공간을 채우는 용도
- 히어로 영역에 소셜 프루프나 기대감 유발 요소가 없어 "왜 해야 하지?"가 느껴지지 않음
- "약 2분 · 5단계 테스트" 텍스트가 너무 작고 (14px, #9B9BB8) 강조가 안 됨 — 이 정보는 신뢰를 높이는 중요한 요소인데 묻혀있음
- CTA 버튼에 onMouseEnter/onMouseLeave hover만 있고 active (탭 누름) 피드백이 없어 모바일에서 반응이 없는 것처럼 느껴짐

**개선 방향:**
- 플로팅 이모지 대신 결과 카드의 미리보기 썸네일이나 "이미 X명이 테스트했어요" 형태의 사회적 증거 추가
- 테스트 완료 후 어떤 결과를 받는지 기대감을 주는 작은 힌트 (예: 점수 배지, 결과 타입 예시)

## 2. 반려동물 정보 (PetInfo)

**문제점:**
- 반려동물 종류 버튼에 `position: absolute`로 체크 배지를 달았는데, 버튼에 `position: relative`가 없어서 배지가 카드 밖으로 overflow될 수 있음 (버그)
- 나이 입력 필드에 숫자를 입력하면 단위(살/개월)가 옆에 분리되어 있어서 입력값과 단위가 시각적으로 한 덩어리처럼 보이지 않음
- 건강 상태 pill들이 flexWrap으로 흘러내리는데 5개 옵션이 불규칙하게 2줄로 나뉘어 레이아웃이 정렬되지 않아 보임
- 에러 메시지가 카드 밖 하단에 표시되어 어떤 입력 항목이 문제인지 직관적으로 연결되지 않음
- 필수 항목 `*` 표시만 있고 페이지 내에 "필수 항목" 설명이 없음

**개선 방향:**
- 종류 버튼에 `position: relative` 추가 (버그 수정)
- 에러 메시지를 문제 있는 입력 필드 바로 아래에 인라인으로 표시
- 건강 상태를 grid 2컬럼 고정 배치로 변경해 레이아웃 안정화

## 3. 사료 정보 (FoodInfo)

**문제점:**
- `Loader2` 스피너 CSS가 `transform: translateY(-50%) rotate(360deg)` — translateY와 rotate를 같이 쓰면 애니메이션에서 translate가 무시되어 아이콘 위치가 틀어질 수 있음
- DB에서 찾았을 때 "✓ DB에서 찾았어요!" 피드백은 좋으나, 해제(X) 버튼이 input 안에 겹쳐있어 터치 타겟이 너무 작음 (16px 아이콘 + 4px 패딩)
- 이 화면이 "2/3 단계"인데 3단계에서 무엇을 하는지 사용자가 모름 — 스텝 헤더에 각 단계 이름이 보이지 않아 진행 상황 파악이 어려움

**개선 방향:**
- 스피너 CSS를 translateY(-50%)를 별도 래퍼로 분리하고 내부 요소에 rotate 적용
- X 버튼 터치 타겟 최소 44×44px 확보

## 4. 설문 (Survey)

**문제점:**
- 프로그레스 바가 1번 문항에서 0%로 시작함 — 사용자는 "아직 시작도 안 했나?" 하고 헷갈림. 1/5이면 20%로 시작해야 함
- `key={qIndex}`로 애니메이션을 트리거하는데, 이전 질문이 slide-out되는 동시에 새 질문이 slide-in되어 두 애니메이션이 겹침. 자연스러운 전환이 아님
- 선택 후 자동으로 다음 질문으로 넘어가는데, 사용자가 실수로 답을 누르면 되돌아갈 방법이 없음 (뒤로 가기를 눌러야 함)
- 선택한 답을 바꾸고 싶어도 이미 넘어간 질문으로 돌아갈 UI 힌트가 없음

**개선 방향:**
- 프로그레스 계산: `((qIndex + 1) / QUESTIONS.length) * 100`으로 변경
- 슬라이드 애니메이션을 순차 처리 (out 완료 후 in 시작) 또는 공통 컨테이너에 overflow hidden + translate 방식으로 교체
- 이전 문항으로 돌아가면 답을 바꿀 수 있다는 인지를 높이는 UI 힌트 추가

## 5. 로딩 (LoadingPaws)

**문제점:**
- 3개 발바닥이 `left: 0px`, `40px`, `80px`으로 가로 배치되어 있는데 이게 "걷는다"는 느낌보다 "순서대로 켜졌다 꺼진다"처럼 보임
- 메시지 4개 중 `{name}` 치환이 2번째 메시지에만 있고 나머지는 고정 텍스트 — 반려동물 이름을 받았는데 활용을 못함
- "AI가 분석하는 중이에요" 부제목이 사이클링 메시지와 중복되는 정보

**개선 방향:**
- petName을 더 많은 메시지에 활용 (예: "잠깐, {name}이가 집사를 평가 중이에요...")
- 발바닥 3개를 세로로 쌓거나 지그재그로 배치해 "발자국" 느낌 강화
- 로딩 시간이 2초 보장인데 실제로 AI 응답이 늦으면 메시지가 계속 반복 — 5초 이상이면 다른 메시지("거의 다 됐어요!") 등 단계적 피드백 추가

## 6. 결과 카드 (Result)

**문제점:**
- `barsVisible` 애니메이션이 최초 마운트 시 한 번만 실행됨 — useEffect가 빈 배열로 한 번만 트리거되므로 이미 결과 페이지에서 새로고침하면 바가 0에서 시작하지 않음
- 헤더 영역(보라 그라디언트)이 카드 전체 높이의 45% 이상 차지 — 정보 밀도에 비해 너무 큰 비중
- "반려동물 시점 메시지" 박스의 `fontStyle: italic`이 한국어에서는 영어와 달리 글자가 기울어지지 않아 italic이 의미 없음 (가독성만 살짝 나빠질 수 있음)
- 이미지 저장, 공유 버튼이 카드 하단에 카드와 충분히 구분되지 않아 "카드에 딸린 버튼"처럼 보임
- 공유 실패 시 `alert()`으로 피드백 — 브라우저 기본 alert이라 디자인과 이질감

**개선 방향:**
- `barsVisible` useEffect deps에 `result`를 추가하거나 key를 통해 컴포넌트 재마운트 유도
- 이탤릭 제거하고 따옴표나 말풍선 스타일 박스로 "반려동물이 하는 말" 표현
- 공유/저장 버튼 영역을 카드와 더 명확히 구분 (배경색 구분, 상단 border 등)
- alert → 인라인 토스트 메시지로 교체

## 공통 이슈

| 항목 | 현재 | 문제 |
|---|---|---|
| **터치 피드백** | hover만 | 모바일에서 버튼 반응 없어 보임 |
| **타이포그래피 스케일** | 13~28px 자유롭게 혼용 | 일관된 타입 스케일 없음 |
| **에러 복구** | 하단 에러 메시지 | 입력 필드와 시각적 연결 없음 |
| **접근성** | aria-label 일부만 | 폼 전체 구조(fieldset/legend) 없음 |
| **빈 상태** | AI 실패 시 폴백만 | 사용자에게 폴백 사용 중임을 알리지 않음 |

## 우선순위

### 즉시 수정 (버그)
- PetInfo 종류 버튼 체크 배지 `position: relative` 누락
- Survey 프로그레스 바 0% 시작 문제

### UX 체감 개선
- Survey 애니메이션 순차 처리
- LoadingPaws 메시지에 petName 활용 강화
- 에러 메시지 인라인화

### 완성도
- alert → 토스트 메시지 교체
- 결과 카드 italic 제거
- 공유/저장 버튼 영역 시각적 분리
