# 제품 목록 페이지 필요성 분석

## 📊 현재 제품 접근 경로

### 1️⃣ 브랜드 중심 탐색
```
/brands
  → 브랜드 선택
  → /brands/[brandName]
  → "제품 목록(n)" 탭
  → /products/[productId]
```

### 2️⃣ 검색 중심 탐색
```
/search
  → "제품(n)" 탭
  → 제품 검색/필터
  → /products/[productId]
```

### 3️⃣ 직접 접근
```
/products/[productId]
```

---

## 🤔 별도 제품 목록 페이지 (`/products`) 필요성

### ❌ 불필요한 경우

#### **현재 서비스가 "브랜드 중심"이면:**

Safe Pet Food의 핵심 가치:
- ✅ 브랜드 투명성
- ✅ SAFI 안전성 점수
- ✅ 브랜드 평가
- ✅ 급여 후기 → 제품 평가

**결론:** 제품은 브랜드의 하위 개념이므로 별도 페이지 불필요

**이유:**
1. 이미 `/search`에서 제품 탭으로 검색 가능
2. 브랜드 상세에서 제품 목록 확인 가능
3. 사용자는 보통 "브랜드 → 제품" 순으로 탐색
4. 제품 수가 많지 않으면 불필요한 페이지

---

### ✅ 필요한 경우

#### **사용자가 "제품 비교" 중심이면:**

예상 사용 시나리오:
- "성견용 사료 전체를 비교하고 싶어요"
- "A등급 제품들만 모아서 보고 싶어요"
- "AAFCO 인증 제품만 비교하고 싶어요"

**구현 시 UI:**
```
/products
  
  필터:
  - 카테고리: 사료/간식/영양제
  - 등급: A/B/C/D/F
  - 브랜드: (멀티 선택)
  - 반려동물: 강아지/고양이
  - 인증: AAFCO/FDA/ISO
  - 가격대: 저/중/고

  정렬:
  - 평점 높은순
  - 추천 많은순
  - 최신순
  - 가격순

  제품 카드:
  - 브랜드명
  - 제품명
  - 등급 배지
  - 평점
  - 추천률
  - 주요 특징
```

**장점:**
- 브랜드 상관없이 제품 비교 가능
- 필터/정렬로 원하는 제품 빠르게 찾기
- 다양한 브랜드의 비슷한 제품 비교

## 💡 추천 의견

### **Option A: 불필요 (현재 상태 유지)** ⭐ 추천

**근거:**
1. Safe Pet Food는 **브랜드 투명성** 중심 서비스
2. `/search`의 제품 탭으로 이미 제품 검색 가능
3. 제품 데이터가 아직 많지 않음 (Mock 단계)
4. 브랜드 상세 → 제품 목록 플로우가 자연스러움

**현재 구조로 충분한 이유:**
```
사용자 여정:

1. 브랜드 신뢰도 확인
   /brands → SAFI 점수, 투명성

2. 제품 선택
   브랜드 상세 → "제품 목록" 탭

3. 제품 평가 확인
   /products/[id] → 급여 후기 기반 실시간 평가
```

---

### **Option B: 필요 (제품 비교 강화)**

**구현 시기:**
- 제품 데이터가 100개 이상 쌓인 후
- 사용자가 "제품 비교" 기능을 요청할 때
- 브랜드 간 제품 비교 니즈가 생길 때

**구현 우선순위:** 낮음 (MVP 이후)
